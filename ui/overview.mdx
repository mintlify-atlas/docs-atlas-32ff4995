The UI system connects `ISetting` instances to on-screen widgets using three cooperating layers: UI elements, binders, and the binder registry. Each layer has a single responsibility, making it straightforward to extend or replace any part.

## Architecture

```
ISetting  ──►  BinderRegistry.FindBinder()
                      │
                      ▼
               ISettingBinder  (selected by Priority + CanBind)
                      │
               Bind(setting, uiElement)
                      │
               ┌──────┴──────────────────┐
               ▼                         ▼
         SetupUI()               SubscribeToUI()
         UpdateUI(value)         (listens for user input)
               │
               ▼
         setting.Changed ──► OnSettingChanged ──► UpdateUI(newValue)
```

When a settings panel needs to display a setting, it:

1. Creates or loads a prefab-based UI item via `ItemFactory`.
2. Asks `BinderRegistry` for a binder that can handle the setting type and UI element type.
3. Calls `binder.Bind(setting, uiElement)` — the binder then owns the connection for its lifetime.
4. Calls `binder.Unbind()` when the panel is torn down.

## ISettingUIElement

`ISettingUIElement` is a **marker interface** — it carries no methods or properties. Its only purpose is to let the type system identify MonoBehaviour components that are part of the settings UI layer.

```csharp
public interface ISettingUIElement { }
```

Every concrete UI item (`SliderItem`, `DropdownItem`, `ToggleGroupItem`, etc.) implements this interface so that `ISettingBinder.CanBind()` can pattern-match against it.

## UIPreference and prefab selection

Each setting carries a `Metadata` object. The `UIPreference` field on that metadata controls which prefab `ItemFactory` loads from Resources.

```csharp
[Metadata(
    displayName: "Master Volume",
    uiPreference: UIPreference.SliderStepper
)]
public class MasterVolumeSetting : FloatSetting { ... }
```

The factory maintains an internal dictionary of `UIPreference → Resources path`. When `CreateAsync()` is called, it looks up that path, loads the prefab asynchronously, and instantiates it under the provided `RectTransform` parent.

See [UI items](/ui/ui-items) for the full path mapping table.
