This guide walks you through installing dependencies, wiring up `SettingsService`, and reading a setting at runtime.

<Steps>
  <Step title="Install dependencies">
    Simple Settings System requires three Unity packages. Install them before copying in the system scripts.

    | Package | How to install |
    |---------|----------------|
    | **Zenject** | Download from the [Modestti/Extenject GitHub releases](https://github.com/Mathijs-Bakker/Extenject/releases) and import the `.unitypackage`. |
    | **TextMeshPro** | Install via **Window > Package Manager > Unity Registry**. |
    | **UniTask** | Add via the Unity Package Manager git URL — see [Installation](/installation) for the exact URL. |

    <Warning>
      Zenject must be installed and its `ProjectContext` asset must exist in a `Resources` folder before you add any `SettingsService` bindings. The system will not compile without Zenject present.
    </Warning>
  </Step>

  <Step title="Bind SettingsService in your ProjectContext">
    Open (or create) your Zenject `ProjectContext` installer and add the two bindings below. `SettingsService` is marked `NonLazy()` so it initializes at startup and auto-registers all settings immediately.

    ```csharp ProjectContext.cs
    [CreateAssetMenu(fileName = "ProjectContext", menuName = "Eql/ProjectContext")]
    public sealed class ProjectContext : ScriptableObjectInstaller
    {
        public override void InstallBindings()
        {
            Container.BindInterfacesAndSelfTo<SimpleResourceService>().AsSingle();
            Container.BindInterfacesAndSelfTo<SettingsService>().AsSingle().NonLazy();
        }
    }
    ```

    <Note>
      `SettingsService` calls `AutoRegister()` inside `Initialize()`. It scans every loaded assembly for concrete `ISetting` implementations and instantiates them automatically — you never need to call `Register()` yourself.
    </Note>
  </Step>

  <Step title="Create your first setting">
    Create a new C# file anywhere in your project. Extend `BoolSetting` and add a `[Metadata]` attribute to describe how it should appear in the UI.

    ```csharp SoundEnabledSetting.cs
    using SimpleSettings;
    using SimpleSettings.Data;

    [Metadata(
        "Sound Enabled",
        "Toggle all in-game sound on or off",
        SettingCategory.Audio,
        "General",
        UIPreference.Toggle,
        0)]
    public sealed class SoundEnabledSetting : BoolSetting
    {
        public override string Key => "audio.sound_enabled";
        public override bool DefaultValue => true;
    }
    ```

    That's all you need. `SettingsService` will discover and register this class automatically at startup.
  </Step>

  <Step title="Retrieve and use the setting at runtime">
    Inject `SettingsService` into any MonoBehaviour or class that Zenject manages, then call `Get<T>()` to retrieve the setting by its concrete type.

    ```csharp AudioManager.cs
    using SimpleSettings;
    using Zenject;

    public sealed class AudioManager : MonoBehaviour
    {
        private SettingsService _settingsService;

        [Inject]
        public void Construct(SettingsService settingsService)
        {
            _settingsService = settingsService;
        }

        private void Start()
        {
            var soundEnabled = _settingsService.Get<SoundEnabledSetting>();
            ApplySound(soundEnabled.Value);
        }

        private void ApplySound(bool enabled)
        {
            AudioListener.volume = enabled ? 1f : 0f;
        }
    }
    ```
  </Step>

  <Step title="Subscribe to changes">
    Each setting exposes a `Changed` `UnityEvent<T>` that fires whenever the value is updated. Subscribe in `Start()` and unsubscribe in `OnDestroy()` to avoid stale listeners.

    ```csharp AudioManager.cs
    private SoundEnabledSetting _soundEnabledSetting;

    private void Start()
    {
        _soundEnabledSetting = _settingsService.Get<SoundEnabledSetting>();

        // Apply current value immediately
        ApplySound(_soundEnabledSetting.Value);

        // React to future changes
        _soundEnabledSetting.Changed.AddListener(ApplySound);
    }

    private void OnDestroy()
    {
        _soundEnabledSetting.Changed.RemoveListener(ApplySound);
    }
    ```
  </Step>
</Steps>

## Complete example

Here is the full `AudioManager` combining every step above:

```csharp AudioManager.cs
using SimpleSettings;
using SimpleSettings.Data;
using UnityEngine;
using Zenject;

public sealed class AudioManager : MonoBehaviour
{
    private SettingsService _settingsService;
    private SoundEnabledSetting _soundEnabledSetting;

    [Inject]
    public void Construct(SettingsService settingsService)
    {
        _settingsService = settingsService;
    }

    private void Start()
    {
        _soundEnabledSetting = _settingsService.Get<SoundEnabledSetting>();
        ApplySound(_soundEnabledSetting.Value);
        _soundEnabledSetting.Changed.AddListener(ApplySound);
    }

    private void OnDestroy()
    {
        _soundEnabledSetting.Changed.RemoveListener(ApplySound);
    }

    private void ApplySound(bool enabled)
    {
        AudioListener.volume = enabled ? 1f : 0f;
    }
}
```
